# Школа нутрициологов «Ресурс» — лендинг

Одностраничный продающий лендинг набора в 6 поток. Старт — 10 сентября 2026.

**Прод:** https://scholnutrition.nutritionforyou.kz

---

## Стек

| Слой | Технология |
|---|---|
| Фреймворк | Astro 6 — zero-JS по умолчанию |
| Стили | Tailwind CSS v4 через `@tailwindcss/vite`, токены в `@theme` |
| Анимации | GSAP 3 + ScrollTrigger, один скрипт на весь сайт |
| Изображения | `astro:assets` — автоматический WebP и `srcset` |
| Раздача | nginx в multi-stage Docker, деплой через Dokploy |

Зависимостей ровно четыре. Ни React, ни Vue, ни клиентского роутера.

## Запуск

```bash
npm install
npm run dev       # http://localhost:4321
npm run build     # сборка в dist/
npm run preview   # проверить собранное
```

Домен задаётся переменной `SITE_URL` — из него Astro запекает `og:image`,
`canonical` и `sitemap.xml`. Без переменной берётся боевой домен:

```bash
SITE_URL=https://staging.example.kz npm run build
```

Docker:

```bash
docker build -t resurs-landing .
docker run -p 8080:80 resurs-landing
```

## Архитектура

```
src/
  assets/img/      исходники — обрабатываются astro:assets
  components/
    ui/            примитивы: Section, Button, Badge, SectionLabel
    *.astro        одна секция = один файл
  data/            program, testimonials, faq, pricing
  layouts/
    Layout.astro   <head>, шрифты и ЕДИНСТВЕННЫЙ GSAP-скрипт
  pages/
    index.astro    только импорты и порядок секций
  scripts/
    ui.ts          бургер, табы, аккордеоны
  styles/
    global.css     @theme-токены и CSS-анимации
reference/         исходный макет и дизайн-система — не собираются
```

Два правила, которые держат систему:

1. **Ни один компонент секции не содержит `<script>`.** Чтобы оживить элемент, вы добавляете класс, а не пишете скрипт. Весь GSAP — в `Layout.astro`, всё поведение — в `scripts/ui.ts`.
2. **Ни одного хекса вне `@theme`.** Смена темы делается в одном месте.

### Анимации

Аккордеоны раскрываются через `grid-template-rows: 0fr → 1fr`, а не через `max-height` — высоту анимирует браузер, JS только вешает класс. Пульс CTA анимирует `opacity` псевдоэлемента, а не `box-shadow`: тень — свойство слоя paint, и бесконечный цикл по ней перерисовывает кнопку каждый кадр.

Бесконечные циклы (`.cta-pulse`, `.hero-glow`) останавливаются за пределами вьюпорта через `IntersectionObserver`.

`prefers-reduced-motion` обработан дважды: в CSS (глушит длительности) и в JS (ранний выход, показывающий весь контент сразу).

## Подводные камни

**`overrides: { vite: "^7.3.6" }` в `package.json` — не украшение.** Без него npm ставит
две копии Vite: Astro 6 тянет 7.3.6, а `@tailwindcss/vite` подтягивает 8.x с другим
rolldown-биндингом. Плагин компилируется под один, исполняется внутри другого, и сборка
падает на `Missing field tsconfigPaths`. Когда Astro перейдёт на Vite 8 — override нужно
поднять или снять, иначе он начнёт мешать.

**Конфликтующие утилиты Tailwind разрешаются порядком в готовом CSS, а не порядком
в атрибуте `class`.** `.border-transparent` эмитится позже `.border-line-strong`, поэтому
`class="border-transparent border-line-strong"` даёт прозрачную рамку. Именно поэтому
`Button.astro` не задаёт цвет рамки в базовых классах — только в каждом варианте отдельно.
Не добавляйте `border-*` в `base`.

**`import.meta.env.BASE_URL` не имеет завершающего слэша**, когда `base` задан как
`/resurs-landing`. Пути к статике собирайте через `withBase()` из `src/lib/url.ts`.
Изображения это не касается — `<Image>` из `astro:assets` подставляет `base` сам.

**`.mod-body` и `.faq-a` обязаны иметь ровно один прямой потомок `<div>`.** На этом держится
раскрытие через `grid-template-rows: 0fr → 1fr` (правило `> * { overflow: hidden }`).
Добавите второй потомок — аккордеон перестанет закрываться.

## Оплата

Кнопки в карточках тарифов открывают модальное окно с формой заказа GetCourse.

| Тариф | Продукт в GetCourse | Цена | id виджета |
|---|---|---|---|
| «Фундамент» | Профессия нутрициолог 2.0 - Фундамент | 149 000 ₽ | `1628151` |
| «Фундамент» + «Система» | Профессия нутрициолог 2.0 - Фундамент + Система | 259 000 ₽ | `1628168` |

Соответствие живёт в `src/data/pricing.ts` рядом с ценой — чтобы цена на карточке
и цена в форме не разъехались. Меняете одно, проверяете второе.

**Скрипт виджета не лежит в разметке.** Он бы отработал на `DOMContentLoaded` и потянул
чужой iframe при каждой загрузке страницы. `src/scripts/payment.ts` подгружает его при
первом открытии окна и вручную посылает событие `StartWidget<domId>` — тот самый крючок,
который GetCourse оставил для отложенной инициализации.

Три вещи, которые легко сломать:

- **`domId` менять нельзя.** Скрипт ищет сам себя через `getElementById(domId)`. Значение
  зашито внутрь файла, который отдаёт GetCourse.
- **Скрипт прописывает своему родителю `overflow: hidden` инлайном.** Поэтому у него
  отдельный контейнер `[data-pay-mount]`, а не карточка тарифа: иначе срезало бы её свечение.
- **`<dialog>` центрируется браузерным `margin: auto`, а preflight Tailwind обнуляет
  `margin` у всех элементов.** Без явного `margin: auto` в `.gc-dialog` окно прилипает
  к левому верхнему углу.

Форма отрисована в `iframe` на домене GetCourse и выполнена в их зелёной палитре.
Перекрасить её под бирюзу школы невозможно — кросс-доменный `iframe` недоступен для CSS.
Окно вокруг сделано нейтральным, чтобы не спорить с ней.

Если браузер не поддерживает `<dialog>`, кнопка открывает ту же форму отдельной вкладкой
(`data-pay-fallback`) — отказ на кнопке оплаты недопустим.

## Известный долг

**84 ScrollTrigger'а на инициализации.** Каждый `once: true`-триггер при создании вызывает
`refresh()`, а тот читает геометрию: суммарно ~3500 layout-чтений, 60–72 мс на десктопе
и ~250–290 мс на мобильном с 4× throttle. Замена 79 одноразовых триггеров на один
`IntersectionObserver` даёт −36% к времени инициализации (замерено: 27.7 → 17.8 мс p50).

Не сделано намеренно: `GUIDE.md` предписывает ScrollTrigger как единый scroll-движок,
цена платится один раз при загрузке, а во время самого скролла ScrollTrigger не делает
ни одного `getBoundingClientRect`. Переписывать проверенную анимационную систему ради
40 мс на старте — плохой размен. Если лендинг пойдёт на слабые Android, вернуться сюда.

Рядом: 7 элементов анимируются дважды (три `.day-marker`, три `.speaker-avatar` и карточка
второго тарифа лежат внутри `.stagger-grid`, чей твин уже гасит их родителя). Уйдёт вместе
с переходом на `IntersectionObserver`.

**Иконки живут в двух идиомах.** `Pain`, `Profession` и `WhyNow` разворачивают цепочки
`{v.icon === 'money' && <svg>…}`, а `Who`, `Format` и `Bonuses` хранят путь строкой
и рендерят через `<Fragment set:html>`. Приведение к одной идиоме и вынос путей
в `src/icons.ts` убирает около 70 строк. Не сделано: правка касается шести проверенных
файлов, а выигрыш — только читаемость.

**Заголовок секции повторён в 13 файлах.** Выносится в `ui/SectionHead.astro` со слотом
(внутри `h2` у `FinalCta` живёт `<em>` с анимированным подчёркиванием) и пропом ширины.
Экономия строк скромная; настоящая польза — невозможность дрейфа вроде трёх заголовков,
которые остались с весом 500, пока остальные тринадцать были 600. Сам дрейф уже исправлен.

## Деплой

Сервер — Dokploy. Пуш в `main` запускает пересборку образа и перезапуск контейнера.

### Настройки в панели

| Поле | Значение |
|---|---|
| Provider | Github → `resurs-landing` |
| Branch | `main` |
| Build Path | `/` |
| Trigger Type | On Push, Autodeploy включён |
| **Build Type** | **Dockerfile** (не Nixpacks) |
| Dockerfile Path | `Dockerfile` |
| Publish Directory | пусто |
| Domains → Host | `scholnutrition.nutritionforyou.kz` |
| Domains → Container Port | `80` |
| Domains → HTTPS | включить, сертификат Let's Encrypt |

**Build Type обязан быть Dockerfile.** Nixpacks попытается угадать способ сборки
и раздачи Astro сам, и потеряет настроенный nginx: gzip, immutable-кэш, страницу 404
и относительные редиректы.

### Домен

Домен нужен **на этапе сборки**: Astro запекает его в `og:image`, `canonical` и `sitemap.xml`.
Он лежит в `Dockerfile` как `ARG SITE_URL` со значением по умолчанию, поэтому образ соберётся
верно даже без настроек в панели. При смене домена поменяйте либо этот дефолт, либо передайте
build-аргумент `SITE_URL`.

Переменная окружения в разделе Environment на это **не влияет**: она попадёт в контейнер
во время исполнения, когда HTML уже собран.

### Локальная проверка образа

Ровно то, что соберёт сервер:

```bash
docker build -t resurs-landing .
docker run --rm -p 8080:80 resurs-landing
```

Что проверять: `/` → 200, `/oferta/` → 200, `/несуществующая` → **404** (а не 200 с главной),
`Cache-Control: no-cache` на HTML и `immutable` на `/_astro/*`, `Location` у редиректов —
относительный.

### Ловушки nginx

- **`try_files $uri $uri/ /index.html`** отдал бы главную с кодом 200 на любой битый адрес.
  У нас `=404` и `error_page 404 /404.html`.
- **`expires 1y` вместе с `add_header Cache-Control`** отдаёт два разных заголовка
  `Cache-Control` в одном ответе. Оставлен только `add_header`.
- **`absolute_redirect off`** обязателен: TLS терминирует Traefik, и без этой директивы
  nginx строил бы `Location: http://…`, уводя человека с https и возвращая лишним прыжком.
- **`add_header` в `location` перекрывает серверные**, а не дополняет. Поэтому заголовки
  безопасности повторены в каждом блоке.
